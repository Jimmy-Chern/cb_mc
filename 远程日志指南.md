# 远程日志获取指南

## 日志地址

每次服务器跑完信号推送，会自动在GitHub Gist生成两份文件：

| 文件 | 内容 | 格式 | 用途 |
|------|------|------|------|
| `cb_signal.json` | 加密交易信号 | AES密文 | QMT调仓用 |
| `cb_log.json` | 明文运行日志 | JSON | **人类/监控用** |

## 获取日志

```python
import requests, json

# 日志是明文的，直接读就行
GIST_URL = "https://gist.githubusercontent.com/Jimmy-Chern/4f3d9550b2c5e874019272b025a58408/raw/cb_log.json"

log = requests.get(GIST_URL, timeout=10).json()

# 看摘要
print(f"运行时间: {log['run_at']}")
print(f"耗时: {log['duration_s']}秒")
print(f"错误数: {len(log['errors'])}")
print(f"警告数: {len(log['warnings'])}")

# 如果有ERROR，打印出来
if log['errors']:
    print("\n⚠️ 发现错误:")
    for e in log['errors']:
        print(f"  [{e['ts']}] {e['msg']}")
        if 'detail' in e:
            print(f"    详情: {e['detail']}")

# 查看完整运行日志（每条都有时间戳）
for entry in log['entries']:
    print(f"[{entry['ts']}] {entry['level']}: {entry['msg']}")
```

## 日志结构

```json
{
  "run_at": "2026-07-15T17:30:00",
  "duration_s": 45.3,
  "total_entries": 25,
  "errors": [
    {"ts": "17:30:05.123", "level": "ERROR", "msg": "FTShare session失败",
     "detail": {"code": 429}}
  ],
  "warnings": [
    {"ts": "17:30:02.456", "level": "WARN", "msg": "12 只因NaN被丢弃"}
  ],
  "entries": [
    {"ts": "17:30:00.001", "level": "INFO", "msg": "启动"},
    {"ts": "17:30:01.234", "level": "OK", "msg": "bond_zh_cov: 1034 只", "detail": {"total": 1034}},
    {"ts": "17:30:03.567", "level": "WARN", "msg": "12 只因NaN被丢弃"},
    ...
  ]
}
```

## 常见ERROR含义

| 错误消息 | 含义 | 严重程度 |
|----------|------|----------|
| `bond_zh_cov拉取失败` | akshare数据源挂了 | 🔴 严重，无信号 |
| `FTShare session失败` | FTShare MCP连不上 | 🟡 中等，正股价回退缓存 |
| `Gist推送失败: 429` | GitHub限流 | 🟡 中等，下次重试 |
| `无有效信号` | 可定价转债不够10只 | 🔴 严重，可能是数据大面积异常 |
| `nan_dropped > 0` | bond_zh_cov部分数据异常 | 🟢 轻微，正常过滤 |

## 定时检查

QMT策略里可以加一段：每天14:00先检查日志，如果有ERROR就告警不交易：

```python
log = requests.get(GIST_URL, timeout=10).json()
if log['errors']:
    print(f"⚠️ 远端{len(log['errors'])}个错误, 今日跳过交易!")
    return  # 不调仓
```
