# 可转债MC定价算法原理详解

## 一、为什么可转债难以定价

可转债 = 债券 + 股票的看涨期权 + 一堆复杂条款。

它不是标准欧式/美式期权，因为有三个中国特色条款：
1. **强赎条款**：正股价连续N天高于转股价的130%，公司有权按面值赎回（逼你转股）
2. **回售条款**：正股价连续N天低于转股价的70%，你有权按面值卖回给公司
3. **下修条款**：回售触发后，公司可能下调转股价（对投资者有利）

这堆条款让解析解（Black-Scholes那种公式）不可能，只能蒙特卡洛。

## 二、三步核心流程

### 第一步：模拟正股价格路径

用几何布朗运动（GBM）生成5000条可能的正股价格路径：

```
S_t = S_{t-1} × exp[(r - σ²/2)/252 + σ/√252 × Z_t]

其中:
  S_t = 第t天正股价格
  r   = 无风险利率（2.5%）
  σ   = 年化波动率（从252天历史日线计算）
  Z_t = 标准正态随机数
```

代码位置：`simulator.py` 第62-76行。

A800 GPU一次生成 5000×756 = 378万个随机数，只需0.01秒。

### 第二步：构建触发变量 Ft 和 Yt

强赎和回售不是"某一天超过就行"，而是"连续N天"。所以用滑动窗口卷积计算运行比例：

```
Ft[t] = 过去call_lb天内 S_daily ≥ 转股价×130% 的比例  (强赎触发度)
Yt[t] = 过去put_lb天内  S_daily ≤ 转股价×70%  的比例  (回售触发度)
```

代码位置：`pricer.py` 第98-120行。

### 第三步：后向归纳（LSM算法核心）

这是整个算法的灵魂。从到期日开始，倒着算到第0天。

#### 3.1 月频行权日期

不是每天检查，而是每隔63个交易日（约3个月）设一个"决策点"。这样756天只有12个决策点，加速60倍，定价误差<1%。

#### 3.2 到期日（T = 到期日）

持有者有两个选择：转股 or 拿赎回价，取大的：

```
V[T] = max(转股价值, 赎回价)
转股价值 = (面值/转股价) × 正股价[T]
```

代码位置：第147-150行。

#### 3.3 往前推（T-3月, T-6月, ... 直到第0天）

在每个决策点t，持有者面临三种情况：

**情况1：强赎触发了（Ft ≥ 阈值）**

公司说了算。如果赎回价 ≥ 转股价值，投资者被迫接受赎回价（K_call）。否则，投资者会立刻转股拿转股价值。

```python
if Ft ≥ pF:
    if K_call ≥ 转股价值:
        被迫赎回 → V[t] = K_call
    else:
        转股 → V[t] = 转股价值
```

**情况2：回售触发了（Yt ≥ 阈值）**

投资者说了算。以概率p_downward（80%），公司会下修转股价来避免回售：

```
新转股价 = max(正股价×0.85, 原转股价×0.85)
转股价值(新) = 面值/新转股价 × 正股价

投资回报 = max(回售价, max(转股价值(新), 继续持有价值))
                                                     ↑
                                            这个需要LSM回归来估计
```

**情况3：什么都没触发**

投资者自己决定：转股 vs 继续持有？

转股价值好算：`面值/转股价 × 正股价[t]`

继续持有价值不能直接算——它是未来所有可能决策的期望折现。这就用到了**最小二乘回归（LSM）**。

#### 3.4 LSM回归：估计"继续持有"值多少钱

LSM的核心思想："只对'正股涨了，转股有利可图'的路径做回归，其他路径反正不会转股。"

**区间划分**：按正股价格分成4个区

| 区间 | 条件 | 名字 | 含义 |
|------|------|------|------|
| Π₁ | S > 转股价×130% | 股权区 | 深度实值，大概率转股 |
| Π₂ | 转股价 < S ≤ 转股价×130% | 平衡股权区 | 浅度实值 |
| Π₃ | 转股价×70% < S ≤ 转股价 | 平衡债权区 | 浅度虚值 |
| Π₄ | S ≤ 转股价×70% | 债权区 | 深度虚值，纯债 |

**回归方程**：在每个区间内用9个基函数拟合继续持有价值

```
y = 下一步价值 × 折现因子
ŷ = θ₁·S + θ₂·S² + θ₃·F + θ₄·F² + θ₅·Y + θ₆·Y² + θ₇·S·F + θ₈·S·Y + θ₉·F·Y
```

9个基函数：{S, S², F, F², Y, Y², SF, SY, FY}

其中 S=正股价, F=强赎触发度, Y=回售触发度。

用最小二乘法求解 θ：
```
X = [5000个样本 × 9个基函数]
y = [5000个样本的折现后价值]

XᵀX · θ = Xᵀy
θ = (XᵀX)⁻¹ · Xᵀy
```

代码位置：`pricer.py` 第242-292行。

**为什么分4个区间？**
不同价格区间的转债行为完全不同：
- 股权区：持有者行为像股票持有者，价格弹性接近1
- 债权区：持有者行为像债券持有者，价格弹性接近0
- 中间两个区需要分别拟合

#### 3.5 决策

```
if 没触发任何条款:
    if 转股价值 > ŷ:  # 回归告诉我们继续持有不如现在转股
        转股 → V[t] = 转股价值
    else:
        继续持有 → V[t] = ŷ
```

### 第四步：算价格

所有路径的所有决策做完了，V[0]就是最终的MC价格：

```
价格 = 所有5000条路径的V[0]的平均值 × 折现到第0天的因子
```

## 三、GPU加速在哪

1. **路径生成**：`torch.randn(5000, 756)` 在GPU上并行生成378万个随机数
2. **滑动窗口卷积**：`torch.nn.functional.conv1d` 在GPU上并行处理5000条路径
3. **回归求解**：`torch.linalg.solve(XᵀX, XᵀY)` 用cuSOLVER加速9×9矩阵求逆
4. **向量化决策**：全部分支用`torch.where`并行处理5000条路径

单只转债定价：5000路径 = 0.15秒。

## 四、折价率选股

```
折价率 = (模型价 - 市场价) / 市场价

折价率 > 0  → 市场低估了，买
折价率 < 0  → 市场高估了，不买
```

每天对所有322只转债MC定价，按折价率排序，买最高的10只，等权持仓。

## 五、完整定价流程示意

```
输入：一只转债的参数
  ├─ 转股价: 13.08
  ├─ 正股价: 8.20 (当前)
  ├─ 波动率: 0.28
  ├─ 市场价: 120.48 (实盘)
  └─ 到期日: 3年

    ┌─────────────────────────┐
    │ 生成5000条GBM路径        │
    │ S₀=8.20, σ=0.28         │
    │ ┌──┬──┬──┬──┬──┬──┐     │
    │ │8.2│8.5│9.1│..│..│..│   │
    │ │8.2│7.9│6.3│..│..│..│   │  ← 5000行 × 756列
    │ │8.2│8.1│8.8│..│..│..│   │
    │ └──┴──┴──┴──┴──┴──┘     │
    └─────────┬───────────────┘
              ↓
    ┌─────────────────────────┐
    │ 计算触发变量 F_t, Y_t     │
    │ F_t = 强赎触发度          │
    │ Y_t = 回售触发度          │
    └─────────┬───────────────┘
              ↓
    ┌─────────────────────────┐
    │ 后向归纳 (每63天一步)     │
    │                         │
    │ T=756天: V=max(转股,赎回) │
    │ T=693天: LSM回归 → 决策   │
    │ T=630天: LSM回归 → 决策   │
    │   ...                   │
    │ T=0天:   最终价格 V₀      │
    └─────────┬───────────────┘
              ↓
    ┌─────────────────────────┐
    │ V₀ = avg(5000条路径折扣)  │
    │ = 126.4 元               │
    │                         │
    │ 折价率 = (126.4-120.48)/│
    │          120.48          │
    │ = +4.9%                  │
    │                         │
    │ ← 低估！买入              │
    └─────────────────────────┘
```
